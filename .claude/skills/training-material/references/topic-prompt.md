# プロンプトエンジニアリング

## 目的

効果的なプロンプトの書き方を習得する

## 経営層向け重点

- プロンプトの重要性（出力品質との関係）
- 基本的な使い方のデモ
- 社内展開時のポイント

## 管理職向け重点

- プロンプトの基本構造
- 実践的なテクニック
- 部下への指導方法

## 必須コンテンツ

### 1. プロンプトの基本構造

```
[役割] あなたは○○の専門家です
[文脈] 以下の状況で...
[指示] ○○してください
[出力形式] 箇条書きで/表形式で
[制約] 300文字以内で
```

### 2. 効果的なプロンプトの5原則

| 原則 | 説明 | 例 |
|-----|-----|-----|
| 明確で具体的 | 曖昧さを排除 | ×「良い文章を」→○「ビジネスメールを敬語で」 |
| 文脈を提供 | 背景情報を含める | 「新規顧客向けの」「社内向けの」 |
| 出力形式を指定 | 期待する形を明示 | 「箇条書き5点で」「表形式で」 |
| 段階的に依頼 | 複雑なタスクは分割 | まず概要→次に詳細 |
| 反復と改善 | 結果を見て調整 | 「もう少し簡潔に」「具体例を追加」 |

### 3. よくある失敗パターン

| 失敗 | 問題 | 改善 |
|-----|------|-----|
| 指示が曖昧 | 「いい感じに」 | 具体的な条件を明示 |
| 情報不足 | 前提を省略 | 必要な背景を提供 |
| 長すぎる | 要点が不明 | 重要な指示を絞る |
| 矛盾した指示 | 「短く詳しく」 | 優先順位を明確に |

### 4. 業務別テンプレート

**会議議事録の要約**
```
以下の会議メモを要約してください。
- 形式: 決定事項、アクションアイテム、次回予定の3セクション
- 各項目は箇条書きで簡潔に
- アクションアイテムには担当者と期限を明記

[会議メモ]
```

**ビジネスメール作成**
```
以下の条件でビジネスメールを作成してください。
- 宛先: [相手の立場]
- 目的: [依頼/報告/お礼など]
- トーン: [丁寧/カジュアルなど]
- 含める情報: [箇条書きで]
```

**報告書ドラフト**
```
以下の情報を元に報告書のドラフトを作成してください。
- 目的: [報告の目的]
- 読者: [想定読者]
- 構成: 背景→現状→課題→提案→まとめ
- 長さ: [目安の文字数]

[元情報]
```

## 演習案（ハンズオン用）

1. **演習1（基礎）**: 会議議事録の要約
   - 用意したサンプル議事録を要約
   - プロンプトを改善して精度向上

2. **演習2（応用）**: ビジネスメール作成
   - シナリオに基づきメール作成
   - 異なるトーンで書き直し

3. **演習3（実践）**: 自分の業務で試す
   - 普段の業務タスクをAIで実行
   - 気づきを共有
