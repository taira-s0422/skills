# Webリニューアル提案スキル 検証シミュレーション

## 1. 検証シナリオ

本シミュレーションは、作成した`web-renewal-proposal`スキルが意図通りに機能するかを検証するために実施します。以下の架空のクライアントシナリオに基づいて、スキルに定義されたワークフローを段階的に実行します。

- **クライアント:** 株式会社ABC製作所
- **業種:** 中小企業の製造業（BtoB）
- **現状:**
    - 10年前に作成したWebサイトが古く、デザインが時代遅れ。
    - スマートフォンに対応しておらず、モバイルユーザーからの閲覧が困難。
    - Webサイトからの問い合わせは月に1件あるかないか。
    - 新規顧客開拓は、主にテレアポや既存顧客からの紹介に依存している。
    - 営業担当者からは「サイトが古くて顧客に見せられない」という声が上がっている。
- **依頼内容:** Webサイトをリニューアルして、Webからの問い合わせを増やし、新規顧客開拓の柱としたい。

## 8. フェーズ4: 提案書の作成 (Proposal Gate) シミュレーション

`SKILL.md`のフェーズ4のプロセスと、調査済みの「決裁が通りやすい8つの構成要素」に基づき、提案書の作成をシミュレートします。

### 4.1 構成の決定 (シミュレーション)

以下の8つの構成で提案書を作成します。

1.  **サイトリニューアルの概要** (サマリー)
2.  **リニューアルの背景と目的 (KGI)**
3.  **現状の課題分析 (自社・競合・データ)**
4.  **ターゲット設定とペルソナ**
5.  **施策内容とリニューアル後の目標 (KPI)**
6.  **実装・運用に関する検討項目**
7.  **予算・スケジュール**
8.  **制作体制**

### 4.2 各セクションの執筆 (シミュレーション)

これまでのフェーズのアウトプットを各セクションに落とし込みます。

- **1. 概要:** KGI、課題、中心施策（オンライン見積もりツール）、概算予算・期間を1枚にまとめる。
- **2. 背景と目的:** テレアポ依存からの脱却と、Webチャネル確立の必要性を訴求。KGIとして「月間問い合わせ10件」を明記。
- **3. 現状分析:** 営業担当者の声、競合の強み、スマホ未対応による機会損失（データ）を提示し、「現状維持のリスク」を強調。
- **4. ペルソナ:** 鈴木部長の具体的な課題と潜在ニーズを提示し、「誰のためのリニューアルか」を明確にする。
- **5. 施策とKPI:** 「オンライン簡易見積もりツール」を中核施策として紹介。各施策がどのように課題を解決し、KPI達成に貢献するかを論理的に説明。
- **6. 実装・運用:** CMSとしてWordPressを提案。社内で「導入事例」「技術ブログ」を更新できる体制を構築することを提案。
- **7. 予算・スケジュール:** 制作費と保守・運用費を分けて提示。バッファを含んだ現実的なスケジュールを引く。
- **8. 制作体制:** 社内担当者と制作会社の役割分担を明確にする。

### 4.3 予算とスケジュールの提示 (シミュレーション)

| 項目 | 金額（概算） | 備考 |
| :--- | :--- | :--- |
| **初期費用（制作費）** | 350万円 | デザイン、コーディング、CMS構築、簡易見積もりツール開発費を含む |
| **月額費用（保守・運用）** | 5万円 | サーバー・ドメイン費、CMS保守、軽微な修正対応 |

- **スケジュール:** 承認後、約5ヶ月で公開予定。

### フェーズ4 アウトプット (サマリー)

- 決裁者（経営層）が気にる「ビジネス成果」と「費用対効果」に焦点を当てた提案書が完成。
